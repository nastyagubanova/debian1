        Часть2
В журнале отмечаются записи типов: SYSCALL, PATH, EXECVE и др. Тип SYSCALL описывает системный вызов, выполненный процессом.

Отчёт aureport содержит диапозон времени журналов, категории зафиксированных событий и их количество.

auid - пользователь, начавший сеанс
uid - текущий пользователь процесса
euid - эффективный UID
comm - имя команды\процесса
exe - путь к исполняемому файлу

        Часть3
Хронология инцидента:
1. Боб вошёл в систему
2. Боб попытался получить доступ к защищённому файлу
3. Боб попытался просмотреть каталог рут
4. Боб прочитал файл passwd
5. Боб создал подозрительный файл
6. Боб записал содержимое в файл
7. Специалист ИБ расследовал, зафиксировал нарушение и занёс в отчёт

Auditd анализирует системный вызов и флаги открытия файла. Чтение выполняется с флагом только на чтение, а изменение - с флагами записи (O_WRONGLY, O_RDWR, O_CREAT и др.)
